# CODEX_WORKFLOW

Native Minute を Codex で運用するときの実務ルールです。

対象ディレクトリ:
- `/Users/karasawatakahiro/Desktop/native-minute`

目的:
- 1つのフォルダに対して複数チャットを使っても混乱しないようにする
- 実装、不具合修正、レビュー、引き継ぎを整理する
- 同じファイルを複数チャットで同時に触って事故るのを防ぐ

## 1. チャットの種類

### 実装用チャット
- 新機能追加
- Phase ごとの実装
- UI / API / service / migration の実装

### 不具合修正用チャット
- 再現済みバグの修正
- build / lint / typecheck / test failure の修正
- E2E failure の切り分けと最小修正

### レビュー用チャット
- コードレビュー
- 設計監査
- Phase 完了判定
- リスク洗い出し

### 引き継ぎ用チャット
- 現状整理
- 圧縮メモ作成
- 次チャット向けの前提整理

## 2. 役割

### 実装用
- 1つの目的を最後まで進める
- 実装、必要最小限の検証、結果報告まで行う
- 最初に「今回触る主要ファイル」を固定する

### 不具合修正用
- 症状、再現手順、対象ファイルを絞る
- 原因特定と最小修正を優先する
- 仕様変更や広いリファクタは持ち込まない

### レビュー用
- 基本は読み取り専用
- まず問題点、重大度、修正順を出す
- 明示指示がない限り、いきなり実装しない

### 引き継ぎ用
- 変更しない前提で状況をまとめる
- 現在の主導線、壊してはいけない前提、次の自然な着手点を整理する
- 次チャットでそのまま貼れる文面を作る

## 3. 新チャットへ切る条件

### 新チャットへ切るべきとき
- 目的が変わる
  - 例: 実装 → レビュー
  - 例: 新機能 → バグ修正
- 触るファイルの中心が変わる
- ひとつのチャットが長くなり、前提が散らかってきた
- Phase が切り替わる
- 「今は修正しない、まず監査だけ」のように作業モードが変わる

### 同じチャットを続けてよいとき
- 同じ目的の延長で、同じファイル群を触っている
- レビュー指摘を受けて、そのまま同じ範囲を修正する
- build / lint / typecheck の修正が同一テーマ内に収まる

## 4. 同時並行のルール

### 同時並行してよい条件
- 触るファイルが明確に分かれている
- 片方が読み取り専用である
- 片方が引き継ぎ整理だけで、コード変更しない
- 実装用チャットとレビュー用チャットを並行するが、レビュー側は編集しない

### 明確に禁止する条件
- **同じファイルを複数チャットで同時編集しない**
- 同じ責務の近接ファイルを、別チャットで同時編集しない
  - 例: `route.ts` と対応する `service.ts`
  - 例: `migration` と `types/database.ts`
- 一方のチャットで前提を変えたのに、他方で古い前提のまま編集しない
- 実装チャットどうしで同じ主導線を同時に触らない

### 実務ルール
- あるチャットが触るファイル群は、最初に宣言する
- 途中で対象ファイルが広がるなら、新チャットを検討する
- 不安なら「今このファイルを触っているチャットは1本だけ」に戻す

## 5. チャット名のルール

形式:
- `NM | 種類 | テーマ | 日付`
- または `NM | PhaseX | テーマ | yyyy-mm-dd`

例:
- `NM | 実装 | Phase5 transcription | 2026-03-25`
- `NM | 不具合修正 | Playwright step4 disabled | 2026-03-25`
- `NM | レビュー | Phase2.5 hardening監査 | 2026-03-25`
- `NM | 引き継ぎ | Playwright E2E 現状整理 | 2026-03-25`

## 7. 短い実務版

### 基本
- 1チャット1目的で進める
- **同じファイルを複数チャットで同時編集しない**
- 実装、修正、レビュー、引き継ぎはチャットを分ける
- 触るファイル群は最初に固定する
- 途中で目的や対象が広がったら新チャットへ切り替える

### 使い分け
- 実装: 新機能や Phase 実装
- 修正: 再現済みバグの最小修正
- レビュー: 読み取り専用で問題抽出
- 引き継ぎ: 次チャット用の圧縮メモ作成

### 同時並行
- OK:
  - 実装1本 + 読み取り専用レビュー1本
  - 引き継ぎ整理チャット
- NG:
  - 同じファイルの同時編集
  - migration と types を別チャットで同時編集
  - 同じ主導線を複数チャットで並行修正

### 終了時
- 実装 / 修正チャットの最後:
  - 変更ファイル一覧
  - 何を直したか
  - build / lint / typecheck / test の結果
  - 次に人がやること
- レビューチャットの最後:
  - 問題一覧
  - 重大度
  - 修正順
- 区切るときは必ず引き継ぎメモを残す
